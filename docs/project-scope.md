# 实训项目范围说明

## 1. 项目基本信息

- 项目名称：心理咨询服务预约平台
- 项目定位：面向心理咨询服务场景的预约与管理信息系统
- 实训方向：企业级方向
- 开发模式：单人独立完成
- 技术架构：
  - 后端：Spring Boot 3 + MyBatis + MySQL
  - 前端：Vue 3 + Vite + Element Plus + ECharts
  - 认证方式：JWT
  - 文件服务：阿里云 OSS

## 2. 项目背景

传统的心理咨询预约流程往往依赖人工沟通、表格登记或分散式消息记录，存在预约信息不集中、时间段冲突难控制、咨询过程难追踪、统计分析不直观等问题。本项目面向心理咨询服务场景，设计并实现一套支持多角色协同的预约管理平台，用于提升预约处理效率、规范咨询服务流程，并为管理端提供业务统计支撑。

## 3. 项目目标

本项目的目标不是实现一个简单的预约页面，而是完成一套具备基本企业后台系统特征的业务平台，具体包括：

- 支持管理员、咨询师、普通用户三类角色使用系统
- 支持咨询师档案与可预约时间段维护
- 支持普通用户在线发起预约申请
- 支持预约状态流转、咨询记录补录和反馈评价闭环
- 支持预约数据、咨询数据和评价数据的统计展示
- 在工程实现上体现认证、校验、异常处理、事务控制、测试记录等企业级方向特征

## 4. 目标用户与角色说明

### 4.1 管理员

主要职责：

- 管理咨询师信息
- 管理可预约时间段
- 查看预约与咨询数据
- 查看统计报表

### 4.2 咨询师

主要职责：

- 查看与本人相关的预约信息
- 在咨询完成后补充咨询记录
- 上传咨询附件

### 4.3 普通用户

主要职责：

- 登录系统
- 查看咨询师与可预约时间段
- 发起预约申请
- 查询个人预约记录
- 在咨询结束后提交反馈评价

## 5. 核心业务闭环

本项目围绕以下业务流程展开：

1. 管理员维护咨询师资料
2. 管理员为咨询师配置可预约时间段
3. 普通用户选择咨询师和时间段发起预约
4. 系统校验时间段是否已被占用，避免重复预约
5. 管理端或业务人员处理预约状态
6. 咨询师完成咨询后录入咨询记录，并联动更新预约状态
7. 普通用户对已完成的预约提交反馈评价
8. 管理员查看预约状态分布、咨询师工作量、月度趋势与评价得分统计

## 7. 已具备的代表性能力

结合当前项目现状，已具备以下较完整的实现基础：

- 基于 JWT 的登录认证与接口拦截
- 基于 `schedule_id` 的预约冲突校验
- 预约状态控制与取消逻辑
- 咨询记录补录时使用事务联动预约完成
- 反馈评价与预约状态联动校验
- MyBatis `resultMap + collection` 的一对多查询场景
- 报表数据查询与图表展示
- 前后端分离开发，并支持前端打包后由后端统一托管
