# 登录页样式与登录反馈修正

## 本轮目标

修正登录页输入框视觉不统一的问题，并处理登录成功时出现“成功提示 + 失败提示”同时弹出的异常体验。

## 问题确认

1. 登录页输入框背景
   - 登录卡片本身已经是浅色半透明风格，但输入框仍保留了偏默认的控件底色，视觉上和周围卡片不够统一。

2. 登录成功后出现两条弹窗
   - 登录接口成功后，前端会继续请求 `/login/info`。
   - 但原先在请求该接口之前，没有先把登录接口返回的 token 写入本地。
   - 请求拦截器因此无法携带 token，导致 `/login/info` 先触发一次 401 提示。
   - 随后登录流程仍然继续，所以用户会看到“失败提示 + 登录成功提示”同时出现。

## 已修改内容

1. 登录页输入框样式统一
   - 文件：`frontend/src/views/login/LoginView.vue`
   - 为登录卡片中的 `el-input` 增加更统一的浅色背景和聚焦态样式。

2. 登录流程修正
   - 文件：`frontend/src/views/login/LoginView.vue`
   - 在调用 `/login/info` 之前，先把登录接口返回的 token 写入本地。
   - 保证请求拦截器能够正常带上 token，避免误触发 401 提示。

3. 退出登录提示补充
   - 文件：`frontend/src/layout/MainLayout.vue`
   - 手动点击“退出登录”时增加成功提示，交互更完整。

## 结果

- 登录页输入框与卡片整体风格保持一致。
- 登录成功时不再出现错误弹窗干扰。
- 退出登录时会出现正常提示。
