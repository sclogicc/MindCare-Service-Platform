# 数据库演示数据更新说明

## 本次调整

本次没有改动表结构，主要是把原有初始化数据整体替换为一套新的中文演示数据。

目标：

1. 展示内容统一使用中文
2. 人名尽量不重复
3. 业务状态更自然
4. 数据量比原来更丰富，便于演示分页、统计和状态流转

## 数据规模

- 管理员：1 个
- 咨询师账号：5 个
- 普通用户账号：10 个
- 咨询师资料：5 条
- 可预约时间段：25 条
- 预约记录：25 条
- 咨询记录：9 条
- 反馈评价：7 条
- 附件记录：4 条

## 保留的登录账号风格

为了兼容现有前端演示和说明文档，保留了这些常用账号：

- `admin / 123456`
- `counselor01 / 123456`
- `user01 / 123456`

同时新增了更多账号用于扩展演示。

## 主要特点

- 所有姓名均为中文且尽量避免重复
- 咨询师擅长方向、预约备注、咨询记录、反馈评价均改为中文
- 已完成、待确认、已确认、已取消四种预约状态都有覆盖
- 已评价和待评价并存
- 已填写和待填写咨询记录并存
- 过去时间段用于已完成数据，未来时间段用于待处理数据，更符合业务逻辑

## 脚本位置

- `sql/init_test_data.sql`

## 使用方式

在数据库已经建表完成的前提下，重新执行：

- `sql/init_test_data.sql`

即可用新的中文演示数据覆盖原有测试数据。
