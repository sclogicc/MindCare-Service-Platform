- `高`
- `中`
- `低`

## 3. 填写说明

- `问题描述`：简要描述缺陷现象，避免只写“报错”
- `复现步骤`：尽量写清楚操作路径，便于后续回归
- `修复提交`：填写对应 Git commit 信息或提交摘要
- `修复说明`：写清楚修复思路，便于答辩时说明过程
- `回归结果`：写明回归是否通过，若未通过需继续跟踪

## 4. 当前使用策略

当前阶段优先记录以下类型的问题：

- 预约状态流转异常
- 时间段重复预约问题
- 角色越权访问问题
- 参数校验不足问题
- 数据一致性问题
- 文件上传与附件关联问题
